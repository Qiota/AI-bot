# AI Discord Bot

Это Discord-бот с поддержкой искусственного интеллекта, созданный с использованием Python. Бот способен отвечать на запросы пользователей, использовать веб-поиск для актуальных данных, форматировать ссылки и проверять их валидность.

## Функциональность
- **AI-ответы**: Использует модели (например, gpt-4o-mini, llama-3.1-8b) для генерации ответов на запросы.
- **Веб-поиск**: Автоматически определяет, когда нужны актуальные данные (новости, погода), и использует веб-поиск.
- **Форматирование**: Поддержка Discord Markdown (жирный, курсив, зачеркнутый текст).
- **Обработка ссылок**: Форматирует ссылки в виде `[текст](<URL>)`, удаляет лишние символы (слэши, скобки) и зачеркивает текст невалидных ссылок (статус-код ≠ 200).
- **Хранение контекста**: Сохраняет историю чата с использованием Firebase или локального хранилища.
- **Логирование**: Цветное логирование в консоль и ротация файлов логов с поддержкой UTF-8.

## Требования
- Python 3.10+
- Установленные зависимости (см. ниже)

## Установка
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Qiota/AI-discord.git
   cd AI-discord
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Настройте файл `.env` с переменными окружения (пример ниже):
   ```
   DISCORD_TOKEN=ваш_токен_бота
   FIREBASE_CRED_PATH=E:\...\...\...\serviceAccountKey.json
   DEVELOPER_ID=ваш_ID_в_discord
   FLASK_PORT=5000
   ```
   - `DISCORD_TOKEN`: Токен вашего Discord-бота (получите в Discord Developer Portal).
   - `FIREBASE_CRED_PATH`: Путь к файлу JSON с учетными данными Firebase (например, `E:\projects\AI discord\src\serviceAccountKey.json`). Укажите полный путь к файлу.
   - `DEVELOPER_ID`: Идентификатор разработчика (опционально, для отладки).
   - `FLASK_PORT`: Порт для локального сервера (если используется Flask, по умолчанию 5000).
4. Запустите бота:
   ```bash
   python main.py
   ```

## Зависимости
Список библиотек указан в `requirements.txt`:
- `discord.py` - Для работы с Discord API.
- `g4f` - Интеграция с AI-моделями.
- `firebase-admin` - Поддержка Firebase (опционально).
- `python-dotenv` - Управление переменными окружения.
- `aiohttp` - Асинхронная проверка ссылок.
- `colorama` - Цветное логирование.

## Конфигурация
- Создайте файл `.env` в корневой директории проекта и укажите переменные окружения, как показано выше.
- Настройте `config.py` или используйте `.env` для настройки бота (токен, уровень логирования и т.д.).
- Настройте `database.py` для работы с Firebase или локальным хранилищем.

## Использование
- Упомяните бота (`@Bot`) или ответьте на его сообщение, чтобы получить ответ.
- Примеры запросов:
  - `@Bot Какой сегодня день?` → Возвращает текущую дату.
  - `@Bot Дай новости` → Использует веб-поиск и форматирует ссылки.

## Разработка
- Код находится в `src/`:
  - `client.py` - Основная логика бота.
  - `logging_config.py` - Настройка логирования.
  - `database.py` - Управление контекстом чата.
  - `command_registry.py` - Регистрация команд.
- Добавляйте новые функции, обновляя `system_prompt` в `generate_response`.
